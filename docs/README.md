## 🏗️ Project Organization

SpecMint follows a clean, organized structure:

```
specmint/
├── docs/              # Documentation
├── cmd/               # CLI application entry points
├── pkg/               # Public packages and libraries
├── internal/          # Private application code
├── test/              # Test files and golden datasets
├── .windsurf/         # Windsurf IDE configuration
├── README.md          # Main project documentation
├── LICENSE            # Project license
└── Makefile           # Build and development automation
```

This structure ensures clear separation of concerns and maintainable codebase organization.
